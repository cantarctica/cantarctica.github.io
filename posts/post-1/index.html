WebDev: Standards are meant to be followed:

Browsers will not and cannot agree on standards....
Ever!
(I promise this is somehow relevant)

While writing the HTML and CSS for this site, I slowly began to run into more and more issues where a given feature, even *core* features, required several implementations.

One of these, for example, was the navbar's CSS. While most browsers accept position:sticky;, Safari and any other browsers that run on macOS or iOS instead need position:-webkit-sticky;. That was up until 2019 when Apple finally fully allowed position:sticky; in a release (Safari 13).

Across the web you can again and again see this where major browsers don't include support for features that sites (even popular ones) rely on.

Sometimes, this lack of implementation is deliberate and reasoned, a good example of this is Firefox and WebHID. WebHID is an API that gives access to "Human Interface Devices", e.g. keyboards. Mozilla has deliberately not added it because HID designers don't account for the threat from a malicious website using this

The issue though, with all of this, is that code very quickly becomes messy as you have to append more and more extra lines for compatibility.

"But is any of this worth it for a 1 or 2% greater market share access?"

Probably not.
